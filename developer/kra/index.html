Developer - KRA

Developer KRA (Key Result Areas) helps to measure performance of Software Development Team objectively!

Developer KRA has various KPI - Key Performance Indicators.

WHAT

  • Result => Generic

    Areas that directly impact the final result.

  • Product/Project/Service => Customers

    Resources that directly impact the final product.

  • Growth => Organizations/Institutions

    Plans that directly impact the final growth

  • Goals => Individuals

    Dreams that directly impact the final goals

  • Process => All

    Roles that directly impact the final process.

How to find those areas?

  • 01. Define Objectives

    When you're really passionate about your job, you can change the world.

  • 02. List Activities

    List activities performed by each role - Customer, Organization and Individuals

  • 03. Analyze Impacts

    Analyze impacts of identified activities on the final result.

  • 04. Prioritieze/Filter Activities

    Prioritieze activities. If required, filter a few activities which are less important.

  • 05. Define KPI

    For each activity, define KPIs and make sure each KPI is aligned with definition of DONE for an activitity.

  • 06. Define Metrics

    List down to metrics to evaluate a KPI. A KPI may have multiple metrics.

  • 07. Define Process

    For measuring metrics, define process that will be helpful to evaluate KPIs of each KRA.

  • 08. Evalute and Enhance

    Evaluate defined process and measure with aligned expectations. If required, enhance the process framework.

KRA - Developer Role

Developer KRA includes programming, learning and adaptability of developer.

  • Coding: LOC, Coding Standard Adherence
  • Programming: Maintainability, Reusability, Testability
  • Documentation: Comment vs Self Documenting Code
  • Code Quality: Bugs Count, Critical Bugs, Blocker Bugs
  • Deployment/Delivery: Timely Delivery,[What are] Deliverable Items
  • Estimation: Estimation Accuracy Rate
  • Research: Knowledge Sharing (Blog, Presentationm), PoC/Demo

KPI: Coding - LOC

Average LOC per day during project duration.

Measurement varies based on technology, experience and excellence.

  • How to Setup?

    Git Blame Tool
    Repository Analysis Tool

  • How to Measure?

    Daily

  • Examples

    100-500 LOC scores 100%

KPI: Coding - Standards

Define important standards technology-wise.

Average broken standards per day based on LOC.

  • How to Setup?

    FxCop
    Visual Studio Code Analysis
    Resharper

  • How to Measure?

    Daily

  • Examples

    10 validations per N LOC scores 100%

KPI: Programming - Reusability

High Cohesive and Low Coupled Coding Practices

Review on each delivery. Contributions in Library, Framework and Open Source Projects.

  • How to Setup?

    nDepend
    Manual Review

  • How to Measure?

    Quarterly Manual Review

  • Examples

KPI: Programming - Testability

Measure Code Coverage

    NUnit
    XUnit

  • How to Measure?

    Daily, Test Results and Code Coverage

  • Examples

KPI: Testing - Bugs

Total# of Bugs, Bug Re-Opened Rate

Critical vs Blocker Bug Rates

  • How to Setup?

    Project Management Tools - TFS, JIRA
    Bug Tracking Tools - Mantis

  • How to Measure?

    Daily Review
    Total# of Bugs
    Bug Re-Opened Rate
    Critical vs Blocker Bug Rates

  • Examples

    0-2 Critical Bugs scores 100% 3-6 Critical Bugs scores 70% 7-10 Critical Bugs scores 50% >10 Critical Bugs scores 10%

KPI: Delivery - Time

Estimation Accuracy Rate

Ball-Park Estimation to Actual Estimation

  • How to Setup?

    Project/Module Estimation - Project Management Tools

  • How to Measure?

    Each Delivery review
    Estimation vs Actual Time Difference

  • Examples

    1 week extend scores 100% 2-3 weeks scores 70% 4-6 weeks scores 50% >6 weeks scores 10%

KPI: Delivery - Deliverables

Ensures all deliverables are delivered

Code Checkin, Deployment, Requirement Docs and Release Note

  • How to Setup?

    Send an email to Stack Holders/Product Owners

  • How to Measure?

    Each Delivery review
    Stack Holders'/Product Owners' Feedback

  • Examples

    Release Note missing/improper scores 50% Forgot to tag version control scores 50% Forgot to send test results score 50% 0-2 missing requirements score 100% >2 missing requirements score 50%

KPI: Estimation - Accuracy Rate

Ensures Accuracy of Estimations

  • How to Setup?

    Estimation Back Tracking, Expert Estimation

  • How to Measure?

    Each Delivery review
    Estimation Back Tracking

  • Examples

    0-10% Accuracy Deviation scores 100% 11-20% Accuracy Deviation scores 70% 21-30% Accuracy Deviation scores 50% >30% Accuracy Deviation scores 30%
